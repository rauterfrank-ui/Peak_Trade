## Wie das Mapping funktioniert

1. Regeln stehen in `config/ops/docs_truth_map.yaml` (`rules[]`).
2. Jede Regel hat `sensitive` (Pfade) und `required_docs` (Pfadliste).
3. Das Skript ermittelt ge√§nderte Dateien via `git diff <base>...HEAD` und pr√ºft pro Regel:  
   **Wenn** mindestens ein `sensitive`-Pfad ge√§ndert wurde **und** **keine** der `required_docs` ge√§ndert wurde ‚Üí **Fehler** (Exit 1).

- Endet ein Eintrag in `sensitive` mit `/`, gelten alle Repo-relativen Pfade **unter** diesem Pr√§fix.
- Sonst exakter Pfad.
