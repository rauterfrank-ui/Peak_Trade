## Wie das Mapping funktioniert

1. Regeln stehen in `config/ops/docs_truth_map.yaml` (`rules[]`).
2. Jede Regel hat `sensitive` (Pfade) und `required_docs` (Pfadliste).
3. Das Skript ermittelt geĂ¤nderte Dateien via `git diff <base>...HEAD` und prĂĽft pro Regel:  
   **Wenn** mindestens ein `sensitive`-Pfad geĂ¤ndert wurde **und** **keine** der `required_docs` geĂ¤ndert wurde â†’ **Fehler** (Exit 1).

Pfad-Matching:

- Endet ein Eintrag in `sensitive` mit `/`, gelten alle Repo-relativen Pfade **unter** diesem PrĂ¤fix.
- Sonst exakter Pfad.
